# Client data

Source material for this project.

## What to put here

- The problem the client wants solved, in their own words.
- Domain notes: how the client's business or system works today.
- Requirements, constraints, and anything explicitly out of scope.
- The intended tech stack, if the client has one in mind.
- Reference material: specs, exports, sample data, screenshots.

Markdown or plain text is easiest to read. PDFs and spreadsheets are fine too.

## What not to put here

- Credentials, API keys, or anything else secret.
- Personal data about the client's customers.
