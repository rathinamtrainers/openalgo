# Iteration 01 — Manual Test Cases

These are the checks you run with your own hands before the automated suite in `04_test_automation.md` runs them for you forever, and before `05_deployment_guide.md` puts the service on a host. The automated tests prove the code behaves; these prove the *system* behaves — that the service starts against a real OpenAlgo, that the kill switch you throw from a terminal actually stops the desk, and that the journal you open in `sqlite3` really refuses to be edited.

## How to run these

Work in a scratch state directory so nothing you do here touches a deployed journal, and point the desk at a local OpenAlgo with a live broker session:

```bash
cd strike_desk
export STRIKE_DESK_OPENALGO_API_KEY='<the key from /apikey>'
export STRIKE_DESK_STATE_DIR="$PWD/.state-manual"
export STRIKE_DESK_TICK_INTERVAL_SECONDS=60
export STRIKE_DESK_LOG_LEVEL=DEBUG
rm -rf "$STRIKE_DESK_STATE_DIR"
uv sync
```

Keep three terminals open with that environment exported: one running `uv run strike-desk run`, one for CLI commands, and one for `sqlite3 "$STRIKE_DESK_STATE_DIR/strike_desk.db"`.

Several cases need the market open or closed on demand. Rather than waiting for a session, override the gate through configuration: widening `STRIKE_DESK_NO_TRADE_WINDOWS` to cover the current time forces a block, and clearing it (`STRIKE_DESK_NO_TRADE_WINDOWS=00:00-00:01`) removes that gate. The exchange window itself comes from OpenAlgo, so cases MT-02 and MT-03 are best run during and outside real NFO hours respectively — a weekend afternoon gives you MT-03 for free.

Nothing in this slice calls a language model, so every case below is fully deterministic: the same inputs must give byte-identical outcomes and reason codes on every run. If a case ever produces a different reason code from the one written here, that is a defect, not model variance.

## The cases

| ID | Title | Preconditions | Steps | Expected result | Covers |

| **MT-01** | Service starts and takes its first tick | OpenAlgo up with a valid broker session; empty state dir; NFO open; no no-trade window active | 1. `uv run strike-desk run`. 2. Watch the log for one cadence interval. | Startup line names the index, cadence, a `ps-` prompt-set version and `roles=()`. Within one interval a line reads `tick <uuid> -> decline/specialist-unavailable in NNms`. `strike-desk journal` shows exactly one row whose sentence names the missing `regime` specialist. | AC-1, AC-4, AC-11 |
| **MT-02** | Exactly one decision per trigger | MT-01 passed; service running | 1. Note the current row count from `strike-desk status`. 2. `uv run strike-desk tick-now`. 3. Re-check `status`. | The count increases by exactly one. The new row's `trigger` column is `manual`, and every column is populated — no NULL outside `regime_label` / `regime_confidence`. | AC-1 |
| **MT-03** | Market closed produces no decision | Run on a weekend or after 15:30 IST; service running | 1. Let one cadence interval pass. 2. `strike-desk status`, then query `traces`. | Log reads `tick skipped: market-closed (...)`. Zero new `decisions` rows. Exactly one new `strike_desk.tick` span whose `attributes_json` contains `"tick.skipped": true` and `"gate.blocked_by": "market-closed"`. Heartbeat still advances. | AC-2 |
| **MT-04** | No-trade window blocks the tick | Service stopped | 1. Set `STRIKE_DESK_NO_TRADE_WINDOWS` to a range covering the current time. 2. Start the service, wait one interval. | Log reads `tick skipped: no-trade-window (inside HH:MM–HH:MM)`. Zero new decision rows; a skipped span names `no-trade-window`. | AC-2 |
| **MT-05** | Expiry-day cutoff blocks the tick | Run on a Tuesday after 14:00 IST, or set `STRIKE_DESK_EXPIRY_WEEKDAY` to today's weekday and `STRIKE_DESK_EXPIRY_CUTOFF` to a time already passed | 1. Start the service, wait one interval. | Log reads `tick skipped: expiry-cutoff (expiry day, past HH:MM)`. Zero new decision rows. | AC-2 |
| **MT-06** | Open position turns the tick into a hold | A non-zero NIFTY NFO position exists in OpenAlgo's position book (open one in sandbox/analyzer mode if needed); market open | 1. `uv run strike-desk tick-now`. 2. `strike-desk journal`. | One new row with outcome `hold` and reason `position-open`; the sentence names the held symbol(s). The trace for that tick contains `tick.plan`, `tick.decide`, `tick.persist` and **no** `tick.consult` span. | AC-3 |
| **MT-07** | Kill switch stops the desk within one cadence | Service running; market open | 1. `uv run strike-desk kill --reason "manual test"`. 2. `strike-desk status`. 3. Wait one interval. 4. `uv run strike-desk resume`, wait one more interval. | Status shows `kill switch : ENGAGED` with the reason and timestamp. The next trigger logs `tick skipped: kill-switch (... manual test)` and writes no decision row. After `resume`, status shows `released` and the following trigger produces a decision row again. | AC-7, AC-2 |
| **MT-08** | Overlapping triggers are skipped, not queued | Service running; market open | 1. Temporarily set `STRIKE_DESK_OPENALGO_TIMEOUT_SECONDS=30` and point `STRIKE_DESK_OPENALGO_BASE_URL` at an unresponsive port (e.g. `http://127.0.0.1:5999`) so a tick stalls. 2. While a tick is in flight, run `uv run strike-desk tick-now` twice in quick succession. | At most one decision row appears for the in-flight tick. Each excess trigger emits a `strike_desk.tick` span with `"gate.blocked_by": "overlap"` and `"tick.skipped": true`. No trigger is executed after the fact. | AC-6 |
| **MT-09** | Unreachable OpenAlgo declines on data quality | Service stopped | 1. Point `STRIKE_DESK_OPENALGO_BASE_URL` at a port with nothing listening. 2. Start the service. | Startup fails fast at the ping with a clear transport error and a non-zero exit — the desk refuses to schedule against a substrate it cannot reach. | AC-13 |
| **MT-10** | OpenAlgo failing mid-session declines on data quality | Service running and ticking normally; market open | 1. Stop the OpenAlgo process. 2. Wait one cadence interval. 3. Restart OpenAlgo and wait one more. | While OpenAlgo is down, the gate blocks with `calendar-unavailable` or, if the calendar was cached, the tick records `decline` / `data-quality`. Either way **zero** rows show outcome `enter` or `hold`. When OpenAlgo returns, ticking resumes without a restart. | AC-13, AC-5 |
| **MT-11** | The journal refuses to be edited | At least one decision row exists | 1. `sqlite3 "$STRIKE_DESK_STATE_DIR/strike_desk.db" "UPDATE decisions SET outcome='enter';"`. 2. Repeat with `DELETE FROM decisions;`, `UPDATE traces ...`, `DELETE FROM traces;`. | Every statement fails with `Error: decisions is append-only` or `Error: traces is append-only`. Row counts are unchanged afterwards. | AC-9 |
| **MT-12** | Every tick leaves a complete, connected trace | At least one executed (not skipped) tick | 1. Take a `trace_id` from `strike-desk journal`. 2. `SELECT name, parent_span_id, duration_ms, status FROM traces WHERE trace_id='<id>' ORDER BY started_at_utc;` | Rows for `strike_desk.tick` (no parent) plus `tick.plan`, `tick.decide`, `tick.persist`, and `tick.consult` when the book was flat. Every child's `parent_span_id` resolves to a span in the same trace. The decision row's `trace_complete` is `1` and its `trace_id` matches. | AC-10 |
| **MT-13** | The API key never reaches the journal, traces or logs | The service has run at least one tick and one failed call | 1. `grep -c "$STRIKE_DESK_OPENALGO_API_KEY" <(sqlite3 "$STRIKE_DESK_STATE_DIR/strike_desk.db" .dump)`. 2. Grep the captured service log for the same string. | Both greps return zero matches. Where a payload was captured, the value reads `***redacted***`. | AC-14 |
| **MT-14** | Tick latency is recorded and the budget is enforced | Service stopped | 1. Set `STRIKE_DESK_TICK_BUDGET_SECONDS=0.001`. 2. Start the service during market hours and let one tick run. | One row with outcome `decline` and reason `tick-timeout`, whose sentence names the budget. Restore the budget to 20 and confirm subsequent rows carry a plausible non-zero `latency_ms` in the tens-to-hundreds of milliseconds. | AC-12 |
| **MT-15** | No order path is reachable from tick code | Source checkout available | 1. `grep -rn "placeorder\|placesmartorder\|optionsorder\|basketorder" strike_desk/src/`. 2. Open a Python shell: `uv run python -c "from strike_desk.openalgo_client import READ_ONLY_PATHS; print(sorted(READ_ONLY_PATHS))"`. | The grep returns nothing. The whitelist prints exactly `/api/v1/funds`, `/api/v1/market/timings`, `/api/v1/ping`, `/api/v1/positionbook`. No journal row anywhere shows outcome `enter`. | AC-5 |
| **MT-16** | Prompt-set version changes when a prompt lands | Service stopped; at least one decision already journalled | 1. Note the current `prompt set` from `strike-desk status`. 2. Create `src/strike_desk/prompts/regime.md` containing `---\nversion: v1\n---\nplaceholder body\n`. 3. `strike-desk status` again, start the service, take one tick. 4. Delete the file afterwards. | The `ps-` hash changes. Rows written after the change carry the new `prompt_set_version`, rows written before keep the old one, and both remain in the journal — the audit stamp distinguishes them. | AC-11 |
| **MT-17** | A failing journal fails the tick closed | Service running; market open | 1. Make the journal unwritable: `chmod 444 "$STRIKE_DESK_STATE_DIR/strike_desk.db"` and `chmod 555 "$STRIKE_DESK_STATE_DIR"`. 2. Wait three cadence intervals. 3. Restore permissions and `strike-desk resume`. | Each tick logs `journal write failed ... the tick failed closed` at CRITICAL with a traceback, and no decision is reported as made. After the third consecutive failure the service engages the kill switch itself and logs `kill switch engaged: journal unwritable`. After restoring permissions and resuming, normal ticking returns. | AC-8 |
| **MT-18** | A registered specialist is consulted and scored | Service stopped | 1. In a Python REPL, build a `StrikeDeskService`, register a stub specialist with `role="regime"` returning `SpecialistResult(role="regime", payload={"label": "trending", "confidence": 0.8}, model_version="stub-1")`, then call `service._runner.run_tick("manual")` during market hours. 2. Repeat with `payload={"label": "event-driven", "confidence": 0.9}` and with a `run` that sleeps past the specialist timeout. | Trending/0.8 → `decline` / `specialist-unavailable` naming the missing `strategist` role, with `regime_label='trending'`, `regime_confidence=0.8` and `model_version='stub-1'` on the row. Event-driven → `decline` / `regime-not-tradeable`. Sleeping specialist → `decline` / `specialist-timeout` after roughly `specialist_timeout_seconds`. None produces `enter`. | AC-4, AC-5, AC-11 |

## Coverage

Every acceptance criterion in `01_use_case.md` has at least one case above: AC-1 (MT-01, MT-02), AC-2 (MT-03, MT-04, MT-05, MT-07), AC-3 (MT-06), AC-4 (MT-01, MT-18), AC-5 (MT-10, MT-15, MT-18), AC-6 (MT-08), AC-7 (MT-07), AC-8 (MT-17), AC-9 (MT-11), AC-10 (MT-12), AC-11 (MT-01, MT-16, MT-18), AC-12 (MT-14), AC-13 (MT-09, MT-10), AC-14 (MT-13).

After a full pass, clear the scratch state (`rm -rf "$STRIKE_DESK_STATE_DIR"`) so no test journal is ever mistaken for a real one — and remember that MT-16's prompt file and MT-14's tiny budget must both be reverted before you deploy.
